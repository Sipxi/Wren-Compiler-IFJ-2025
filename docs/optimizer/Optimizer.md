# 🚀 Как работает Оптимизатор (Optimizer Explained)

Этот модуль отвечает за улучшение "Трёхадресного кода" (TAC), который генерирует наш компилятор. Цель — сделать код короче и быстрее, не меняя логику программы.

Мы используем **циклический подход**: мы запускаем все оптимизации по кругу, пока код продолжает меняться. Почему? Потому что одна оптимизация (например, распространение констант) может создать возможности для другой (например, удаление мертвого кода).

Вот 4 техники, которые мы используем:

-----

## 1\. Распространение констант (Constant Propagation)

**Суть:** Если мы знаем, что переменная `x` равна `5`, то везде, где используется `x`, мы можем сразу подставить `5`.

### 💾 Алгоритм (как это работает в коде):

1.  Идём по инструкциям сверху вниз.
2.  Поддерживаем "таблицу значений" (в коде это `PropagatorMap`).
3.  Если видим `x = 5` → запоминаем в таблице, что `x` это `5`.
4.  Если видим `y = x + 2` → смотрим в таблицу. Ага, `x` это `5`. Заменяем инструкцию на `y = 5 + 2`.
5.  **Важно:** Если встречаем `LABEL` (метку) или `JUMP` (прыжок), мы очищаем таблицу. Мы не знаем, откуда мы пришли в эту точку, поэтому старые значения могут быть неверны.

### 📝 Пример:

**До:**

```text
ASSIGN 10 -> a      ; Мы запомнили: a = 10
ADD a, 5 -> $t1     ; Используем 'a'
```

**После:**

```text
ASSIGN 10 -> a
ADD 10, 5 -> $t1    ; Заменили 'a' на 10
```

-----

## 2\. Свёртка констант (Constant Folding)

**Суть:** Если в выражении участвуют только известные числа (константы), зачем считать это во время выполнения программы? Посчитаем это прямо сейчас, во время компиляции\!

### 💾 Алгоритм:

1.  Идём по инструкциям.
2.  Проверяем: это математика (плюс, минус, умножить, соединить строку)?
3.  Проверяем: оба аргумента — это константы (числа или строки)?
4.  Если ДА → Выполняем операцию прямо в Си.
5.  Заменяем сложную инструкцию на простое присваивание результата.

### 📝 Пример:

**До:**

```text
ADD 10, 5 -> $t1    ; 10 и 5 — константы
```

**После:**

```text
ASSIGN 15 -> $t1    ; Компилятор сам посчитал 10+5
```

-----

## 3\. Удаление недостижимого кода (Unreachable Code Elimination)

**Суть:** Инструкции, до которых поток выполнения никогда не доберется, только занимают место. Их нужно удалить.

### 💾 Алгоритм:

1.  Ищем инструкции, которые прерывают поток выполнения: `JUMP` (безусловный прыжок) или `RETURN`.
2.  Если нашли такую инструкцию, начинаем удалять всё, что идет **сразу после неё**.
3.  **Стоп-сигнал:** Перестаем удалять, если встретили `LABEL` (метку). Почему? Потому что на эту метку могут прыгнуть из другой части программы.

### 📝 Пример:

**До:**

```text
JUMP Label_End
WRITE "Этого никто не увидит"   ; <-- Удаляем
ADD 1, 2 -> $t1                 ; <-- Удаляем
LABEL Label_End                 ; <-- Стоп! Сюда прыгают, оставляем.
```

**После:**

```text
JUMP Label_End
LABEL Label_End
```

-----

## 4\. Удаление мёртвого кода (Dead Code Elimination - DCE)

**Суть:** Если мы посчитали переменную, но **никогда** её не использовали, то зачем мы тратили время на вычисления?

### 💾 Алгоритм:

1.  **Проход 1 (Подсчет):** Пробегаем по всему коду и считаем, сколько раз каждая временная переменная (`$t1`) или обычная переменная была прочитана (использована как аргумент).
2.  **Проход 2 (Чистка):** Снова идем по коду.
3.  Если видим инструкцию, которая записывает результат в переменную (например, `ADD ... -> $t1`).
4.  Проверяем счетчик для `$t1`. Если он равен **0** (переменная никому не нужна)...
5.  ...И операция **не имеет побочных эффектов** (это не `CALL` функции, не `WRITE` в консоль), то удаляем инструкцию.

### 📝 Пример:

**До:**

```text
ASSIGN 5 -> a       ; 'a' больше нигде не встречается
ADD 10, 20 -> $t1   ; $t1 используется ниже
WRITE $t1
```

**После:**

```text
                    ; Строка с 'a' удалена
ADD 10, 20 -> $t1
WRITE $t1
```

-----

## 🔄 Как они работают вместе (Синергия)

Магия происходит, когда мы запускаем их в цикле `while(optimization_performed)`.

Представьте такой код:

```text
a = 5
b = 10
c = a + b
return
print(c)
```

1.  **Constant Propagation:** Заменит `a` и `b` в сложении.
    `c = 5 + 10`
2.  **Constant Folding:** Посчитает сумму.
    `c = 15`
3.  **Unreachable Code:** Удалит `print(c)`, так как он после `return`.
4.  **Dead Code Elimination:** Теперь увидит, что `c` (равная 15) вычисляется, но `print(c)` был удален, значит `c` больше нигде не используется. Удалит строку `c = 15`.

**Итог:** Останется только `return`. Программа стала идеальной\!
