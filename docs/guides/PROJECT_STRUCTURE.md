# Структура проекта IFJ-2025

Нельзя сделать красиво -> Я прочитал на елернинге,
что нужно пихать всё в одну папочку, но мы можем пихать их пока в две!
Чтобы было ну хотя бы как-то поудобнее, потом когда будем передавать проект преподам просто пихнем всё обратно в одну папку.

## Текущая структура проекта
```
IFJ-2025/
├── .git/                         # Git метаданные (система контроля версий)
├── LICENSE                       # Лицензия проекта
├── README.md                     # Документация проекта
├── docs/                         # Общие документы
│   └── guides/                   # Руководства для разработчиков
│       ├── C_CONVENTIONS.md      # Соглашения по написанию кода на C
│       ├── GIT_GUIDE.md          # Руководство по работе с Git
│       └── PROJECT_STRUCTURE.md  # Описание структуры проекта
└── prj/                          # Основная папка с исходным кодом
    ├── include/                  # Заголовочные файлы (.h)
    │   └── example.h             # Пример заголовочного файла
    └── src/                      # Файлы с исходным кодом (.c)
        └── example.c             # Пример исходного файла
```



## Описание папок и файлов для новичков

### 📁 Корневая папка (`IFJ-2025/`)
- **LICENSE** - файл лицензии, определяющий права использования кода
- **README.md** - главный файл документации с описанием проекта
- **.git/** - служебная папка Git (НЕ ТРОГАТЬ! Git хранит здесь историю изменений)

### 📁 Документация (`docs/`)
Здесь хранится вся документы проекта:
- **guides/** - руководства и инструкции для разработчиков
  - `C_CONVENTIONS.md` - правила написания кода на языке C
  - `GIT_GUIDE.md` - как работать с системой Git
  - `PROJECT_STRUCTURE.md` - структура проекта (этот файл)

### 📁 Проект (`prj/`)
Основная папка с исходным кодом компилятора:

#### 📁 Заголовочные файлы (`prj/include/`)
Содержит `.h` файлы с объявлениями:
- Функций (что функция делает, но не как)
- Структур данных (типы данных)
- Констант (неизменяемые значения)
- Макросов (сокращения кода)

#### 📁 Исходный код (`prj/src/`)
Содержит `.c` файлы с реализацией:
- Основной логики программы
- Функций, объявленных в заголовочных файлах
- Конкретного кода (как работают функции)

## Рекомендуемая структура для компилятора IFJ

**СГЕНЕРИРОВАНО ИИ**

По мере развития проекта структура должна выглядеть так:

```
prj/
├── include/
│   ├── lexer.h             # Лексический анализатор (разбор токенов)
│   ├── parser.h            # Синтаксический анализатор (разбор грамматики)  
│   ├── semantic.h          # Семантический анализатор (проверка смысла)
│   ├── codegen.h           # Генерация кода (создание исполняемого кода)
│   ├── symtable.h          # Таблица символов (хранение переменных/функций)
│   ├── error.h             # Обработка ошибок
│   └── common.h            # Общие определения и константы
└── src/
    ├── main.c              # Главный файл программы (точка входа)
    ├── lexer.c             # Реализация лексера
    ├── parser.c            # Реализация парсера
    ├── semantic.c          # Семантический анализ
    ├── codegen.c           # Генерация кода
    ├── symtable.c          # Работа с таблицей символов
    └── error.c             # Обработка ошибок
```
